# karabiner-config-builder

A TypeScript config builder for [Karabiner-Elements](https://karabiner-elements.pqrs.org/).

Write Karabiner configuration in TypeScript, then generate a valid `karabiner.json`.

The goal is to make Karabiner configs easier to author, reuse, test, and maintain without writing large JSON files by hand.

## Requirements

- macOS
- Karabiner-Elements
- Node.js
- npm

## Install dependencies

```sh
npm install
```

## Quickstart

By default, the builder loads:

```txt
~/.config/karabiner-config-builder/config.ts
```

and writes:

```txt
./karabiner.json
```

Run:

```sh
npm run build
```

## Basic example

```ts
import { bind, key, profile, rule, setup } from "karabiner-config-builder";

export const config = setup(
    profile(
        {
            name: "Default",
            selected: true,
            virtual_hid_keyboard: {
                keyboard_type_v2: "ansi",
            },
        },

        rule(
            "Basic bindings",

            bind("caps_lock", key("escape")),
            bind("f18", key("spacebar")),
        ),
    ),
);
```

You can also export the config as the default export:

```ts
export default config;
```

## Core concepts

- `setup()` creates the full Karabiner config.
- `profile()` creates a Karabiner profile.
- `rule()` groups bindings and layers.
- `bind()` creates a Karabiner basic manipulator.
- Output helpers like `key()`, `shell()`, `script()`, `app()`, and `url()` create actions.
- Condition helpers like `inApp()`, `fromDevice()`, and `variableIs()` scope bindings.

## Example layer

```ts
rule(
    "Navigation layer",

    layer("caps_lock", {
        tapped: key("escape"),

        bindings: {
            h: key("left_arrow"),
            j: key("down_arrow"),
            k: key("up_arrow"),
            l: key("right_arrow"),
        },
    }),
);
```

## Rule-level conditions

Karabiner stores conditions on individual manipulators, not on rules.

The builder supports rule-level conditions as a convenience by applying them to every manipulator generated by that rule.

```ts
rule(
    {
        description: "Pro Tools bindings",
        conditions: [inApp("com.avid.ProTools")],
    },

    bind("m", key("escape")),
    bind("p", key("spacebar")),
);
```

Rule-level conditions are merged with manipulator-level conditions. Compatible list-based conditions are merged and deduped, and obvious impossible condition combinations are rejected.

See [conditions docs](./docs/conditions.md) for details.

## Common commands

Typecheck:

```sh
npm run check
```

Build the default config:

```sh
npm run build
```

Build test configs:

```sh
npm run build:test:bind
npm run build:test:layer
npm run build:test:conditions
```

Deploy:

```sh
npm run deploy
```

Backup:

```sh
npm run backup
```

Restore:

```sh
npm run restore
```

## Warning

This project generates Karabiner config files. A bad config can break your keyboard behavior until fixed.

Keep backups.

Before deploying a new generated config, it is a good idea to run:

```sh
npm run check
npm run build
```
